
.. index:: File formats

Supported File Formats
======================

+----------------+--------------+-----------------------------------------+
|Source          |File Extension| Description                             |
+================+==============+=========================================+
| :ref:`DICOM`   |\*.dcm        | Dicom CT or RT image files uncompressed |
+----------------+--------------+-----------------------------------------+
| :ref:`MapCheck`|\*.txt        | MapCheck text files                     |
+----------------+--------------+-----------------------------------------+
| :ref:`PTW`     |\*.mcc        | PTW 729 array files only at this stage  |
+----------------+--------------+-----------------------------------------+
| :ref:`IBA`     |\*.opg        | IBA Matrix and StarTrack opg files      |
+----------------+--------------+-----------------------------------------+
| :ref:`Image`   |\*.bmp        | Windows bitmap                          |
+----------------+--------------+-----------------------------------------+
|                |\*.tif,\*.tiff| Tagged Image format files.              |
+----------------+--------------+-----------------------------------------+
|                |\*.jpg,\*.jpeg| Joint Photographic Expert Group files   |
+----------------+--------------+-----------------------------------------+
| :ref:`HIS`     |\*.his        | Raw unstructured data                   |
+----------------+--------------+-----------------------------------------+
| :ref:`XiO`     |None          | Elekta XiO dose plane export text file  |
+----------------+--------------+-----------------------------------------+
| :ref:`iPlan`   |None          | BrainLab iPlan dose plane text file     |
+----------------+--------------+-----------------------------------------+

As the 2D array files have all been reverse engineered there is absolutely no 
guarantee of BeamScheme operating correctly on them. If you have a file that is 
causing BeamScheme to crash or is giving strange results please forward the file 
along with a description of the problem to alanphys@yenzakahle.co.za.
